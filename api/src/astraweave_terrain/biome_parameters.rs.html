1//! Phase 1.6-F.4.B.3.D.3: per-biome terrain parameter system.
2//!
3//! Replaces the legacy `BiomeNoisePreset` (whole-world per-preset
4//! configuration) with a per-`BiomeId` parameter table. Each vertex looks
5//! up its `BiomeId` from the [`crate::biome_lookup::lookup_biome`] function
6//! (D.2), then queries [`BiomeParameters::for_biome`] for the terrain-shape
7//! parameters that apply at that vertex.
8//!
9//! ## Structural change vs F.4.B.3.C and prior
10//!
11//! Before D.3, the user picked a single "Primary Biome" preset and the
12//! engine applied that preset's noise/erosion/scatter parameters to every
13//! vertex of the world. After D.3, the user picks a `WorldArchetype`
14//! (climate envelope; D.5) which shapes the climate field; the climate
15//! field at each vertex resolves to a `BiomeId`; the `BiomeId` resolves
16//! to a `BiomeParameters` table entry. The user does not pick parameters
17//! directly — biomes emerge from climate.
18//!
19//! ## Why some fields are stubbed
20//!
21//! D.3's structural job is the per-vertex parameter lookup. Several
22//! fields on `BiomeParameters` are defined here for API completeness but
23//! their wiring through downstream subsystems is deferred:
24//!
25//! - `mountains_amplitude`: WIRED through D.3b's refactor of
26//!   `WorldGenerator` + `TerrainNoise::sample_components`. Per-biome
27//!   amplitude scaling on the mountain layer.
28//! - `ridge_strength`: DEFINED but not yet wired into the noise
29//!   pipeline. F.4.B.3.E was demoted to this parameter; ridged-multifractal
30//!   integration is a separate noise contribution that requires a follow-up
31//!   tuning campaign. The field is set per biome so future wiring sees
32//!   correct defaults.
33//! - `runevision_config`: DEFINED but not yet wired. F.4.B.3.C's
34//!   per-`BiomeNoisePreset.runevision_enabled` flag flowed through a global
35//!   `NoiseConfig.runevision: Option<RunevisionConfig>`; the new system
36//!   is per-vertex which `TerrainNoise::sample_height` doesn't currently
37//!   support. Mountain-character biomes default to `None` per the
38//!   F.4.B.3.C REGRESS finding (gradient magnitudes pushed the filter
39//!   outside its working range at high amplitudes); this default carries
40//!   forward when wiring lands.
41//! - `erosion_preset`: DEFINED. The legacy `erosion_preset_for_climate`
42//!   maps `ClimateBias` enum values to preset constructors; D.3b will
43//!   route per-biome `ErosionPresetId` through to actual erosion
44//!   simulation calls during `generate_chunk_with_climate`.
45//! - `scatter_density`, `scatter_species_set`, `surface_color_palette`:
46//!   DEFINED but consumed by the scatter and rendering subsystems which
47//!   are downstream of D.3's terrain-pipeline scope. These are forward-
48//!   compatible defaults; D.5+ wires them through.
49//!
50//! ## Why per-biome defaults are conservative
51//!
52//! All 19 biomes default `runevision_config: None`. The F.4.B.3.C lesson
53//! is that opt-in (per-biome) is the safe default — runevision composes
54//! badly with high-amplitude mountain biomes, and bespoke per-biome
55//! tuning to find safe parameters is an investigation campaign, not a
56//! D.3 task. The deferred per-biome runevision tuning (in §4 of the
57//! reframe doc) is the right home for finding biome-specific safe
58//! parameters; until then, every biome opts out.
59//!
60//! The other defaults (mountains_amplitude, ridge_strength) start from
61//! mid-range conservative values that produce visually plausible terrain
62//! across the full archetype × biome cross-product. D.6's Andrew-gate
63//! will identify which defaults need tuning.

69/// Phase 1.6-F.4.B.3.D.3: erosion preset identifier per biome.
70///
71/// Selects one of the existing `advanced_erosion::ErosionPreset`
72/// constructors. The actual `ErosionPreset` struct is large and not
73/// `Copy`-friendly, so `BiomeParameters` stores an ID and resolves to
74/// the full preset only when erosion runs (per-chunk, not per-vertex).
75#[derive(Debug, Clone, Copy, PartialEq, Eq, Hash, Serialize, Deserialize)]
76pub enum ErosionPresetId {
77    /// Default 25K-droplet preset for moderate-erosion temperate climates.
78    DefaultBalanced,
79    /// 35K-droplet preset for mountain-character climates needing more
80    /// erosion to carve dramatic relief.
81    MountainBalanced,
82    /// Arid-climate preset.
83    Desert,
84    /// Wet-climate / coastal preset.
85    Coastal,
86    /// Legacy 100K-droplet aggressive mountain preset (slow; rarely used).
87    Mountain,
88}
89
90impl ErosionPresetId {
91    /// Resolve the ID to a full `ErosionPreset`. Calls the corresponding
92    /// constructor on `advanced_erosion::ErosionPreset`.
93    pub fn resolve(self) -> crate::advanced_erosion::ErosionPreset {
94        match self {
95            Self::DefaultBalanced => crate::advanced_erosion::ErosionPreset::default_balanced(),
96            Self::MountainBalanced => crate::advanced_erosion::ErosionPreset::mountain_balanced(),
97            Self::Desert => crate::advanced_erosion::ErosionPreset::desert(),
98            Self::Coastal => crate::advanced_erosion::ErosionPreset::coastal(),
99            Self::Mountain => crate::advanced_erosion::ErosionPreset::mountain(),
100        }
101    }
102}

165/// Phase 1.6-F.4.B.3.D.3: per-biome terrain parameters.
166///
167/// Looked up per-vertex via [`BiomeParameters::for_biome`] after the
168/// climate field + Whittaker lookup determine the dominant `BiomeId`
169/// at the vertex. Replaces the legacy `BiomeNoisePreset` whole-world
170/// configuration.
171///
172/// All 19 `BiomeId` variants resolve to a populated struct (no panics,
173/// no fallback). See module docs for which fields are wired vs stubbed
174/// in D.3.
175#[derive(Debug, Clone)]
176pub struct BiomeParameters {
177    /// Per-biome multiplier on the mountain layer's noise contribution.
178    /// `1.0` = baseline (use `NoiseConfig::default()`'s mountain
179    /// amplitude as-is); `< 1.0` = damped (rolling biomes); `> 1.0` =
180    /// boosted (alpine biomes). Multiplies the mountain layer's
181    /// per-vertex contribution AFTER continental modulation.
182    ///
183    /// Wired through `TerrainNoise::sample_components` + per-vertex
184    /// combine in `WorldGenerator` (D.3b).
185    pub mountains_amplitude: f64,
186
187    /// Strength of ridged-multifractal noise contribution added to the
188    /// base fBm. `0.0` = no ridges (rolling terrain); `1.0` = full
189    /// ridge contribution. Absorbs F.4.B.3.E (originally a separate
190    /// sub-phase, now a per-biome parameter).
191    ///
192    /// **D.3 status**: DEFINED but not yet wired. Wiring requires adding
193    /// a ridged-multifractal noise source to `TerrainNoise` and routing
194    /// per-vertex contribution through `sample_components`. Defer to
195    /// follow-up tuning campaign.
196    pub ridge_strength: f64,
197
198    /// Per-biome runevision filter configuration. `None` = filter off
199    /// for this biome; `Some` = filter on with these parameters.
200    /// Mountain-character biomes (Alpine, MountainRocky, SnowCap, Scree)
201    /// default to `None` per the F.4.B.3.C REGRESS finding.
202    ///
203    /// **D.3 status**: DEFINED but not yet wired. F.4.B.3.C's flag
204    /// flowed through a global `NoiseConfig.runevision`; per-vertex
205    /// wiring requires `TerrainNoise::sample_height` to accept per-call
206    /// runevision config. Defer to follow-up tuning campaign that also
207    /// finds safe per-biome runevision parameters.
208    pub runevision_config: Option<RunevisionConfig>,
209
210    /// Erosion preset identifier. Resolves to a full
211    /// `advanced_erosion::ErosionPreset` via [`ErosionPresetId::resolve`].
212    ///
213    /// **D.3 status**: DEFINED but legacy `erosion_preset_for_climate`
214    /// path still drives `WorldGenerator::generate_chunk_with_climate`'s
215    /// erosion call. D.3b retains the legacy mapping (it's keyed by
216    /// `ClimateBias` not `BiomeId`); future per-biome erosion routing
217    /// can use this field.
218    pub erosion_preset: ErosionPresetId,
219
220    /// Per-biome vegetation density multiplier. `0.0` = barren; higher =
221    /// denser scatter. Multiplies the global scatter density at scatter
222    /// time.
223    ///
224    /// **D.3 status**: DEFINED but consumed by scatter subsystem
225    /// downstream of D.3's terrain-pipeline scope.
226    pub scatter_density: f64,
227
228    /// Per-biome scatter species set selector.
229    ///
230    /// **D.3 status**: DEFINED, see field docs above.
231    pub scatter_species_set: ScatterSpeciesSet,
232
233    /// Per-biome surface color palette selector.
234    ///
235    /// **D.3 status**: DEFINED, consumed by splat-texturing in the
236    /// rendering subsystem.
237    pub surface_color_palette: SurfaceColorPalette,
238}

477        // F.4.B.3.C REGRESS finding: runevision composes badly with
478        // high-amplitude mountain biomes. Mountain-character biomes

517    #[test]
518    fn phase_1_6_f4_b_3_d_3_alpine_biomes_have_elevated_mountains() {
519        // High-altitude biomes get elevated (above-grassland) mountain
520        // amplitude. Phase 1.6-F.4.B.3.D.5-fix Path B reduced these from
521        // ≥2.0 to the 1.2-1.6 range to fix the explosive radial spike
522        // pattern caused by 2.5×-3.0× multipliers compounding with bootstrap
523        // noise (max real-chunk elevation 1214m vs Target B's 510m design).
524        // The invariant that elevation-overlay biomes have HIGHER amplitude
525        // than the rolling-class baseline (TemperateGrassland at 0.8×) is
526        // what this test now asserts, not a specific 2.0 floor.
527        for &biome in &[
528            BiomeId::Alpine,
529            BiomeId::MountainRocky,
530            BiomeId::SnowCap,
531            BiomeId::Scree,
532        ] {
533            let params = BiomeParameters::for_biome(biome);
534            assert!(
535                params.mountains_amplitude >= 1.0,
536                "{:?} must have mountains_amplitude >= 1.0 (above grassland-class); got {}",
537                biome,
